# Retail Customer & Transaction Analytics

## 📌 Project Overview
This project analyzes day-to-day retail transactions alongside customer demographic data to extract actionable business insights. Using a multi-sheet retail dataset, I performed data merging, exploratory data analysis (EDA), and customer profiling to solve critical business questions regarding regional performance, high-value consumer demographics, and category trends.

## 🛠️ Tech Stack & Tools
- **Language:** Python 3.x
- **Libraries:** Pandas, NumPy, Matplotlib, Seaborn
- **Environment:** Jupyter Notebook

## 📂 Repository Structure
- `retail_customer_analytics.ipynb`: Main notebook containing data cleaning, merging, calculations, and visual charts.
- `README.md`: Project documentation and executive summary.

## 💡 Key Business Questions Addressed
1. **Data Unification:** Merged Customer, Product Hierarchy, and Transaction datasets into a unified profile (`Customer_Final`) using an appropriate join to retain all transacting customers.
2. **Exploratory Data Analysis:** Profiled data structure types, generated five-number summaries for continuous metrics, and built frequency tables for categorical attributes.
3. **Demographic Insights:** Isolated product category popularity mapped across gender segments and the core 25–35 age demographic.
4. **Store & Operational Metrics:** Identified high-performing store formats by sales value vs. quantity, and calculated financial impacts from returns (negative transaction amounts).
